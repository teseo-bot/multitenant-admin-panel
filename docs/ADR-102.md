# ADR-102: Ruteo Dinámico de LLMs y Bypass del Gatekeeper

**Estado:** IMPLEMENTADO
**Fecha:** 2026-04-28

## 1. Contexto
Los modelos LLM estaban hardcodeados en el orquestador y el nodo Gatekeeper se evaluaba incondicionalmente en cada turno, inyectando latencia a sesiones persistentes (SDR/RAG). Además, el nodo de Enrichment tenía un fallo de ruteo devolviendo siempre a SDR.

## 2. Solución
- Se creó `migrations/002_add_llm_config.sql` para parametrizar `llm_config` por Tenant.
- Se inyectó `llmConfig` al Annotation principal del `state.ts`.
- `setupContext.ts` ahora hidrata los prompts y config del LLM en paralelo vía `Promise.all`.
- `graph.ts` fue actualizado con un `addConditionalEdges` de bypass hacia SDR/RAG si `currentRoute` ya existe en el estado, y el retorno de Enrichment fue corregido.
